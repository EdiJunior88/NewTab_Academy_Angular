Execução do aplicativo

Abra o prompt de comando node.js ou seu terminal e acesse seu projeto usando o comando cd e, em seguida, execute o comando ng serve para compilar e executar seu aplicativo.

Abra o navegador e acesse o host local em: http://localhost:4200/

Agora, aprenderemos como o aplicativo do Angular é carregado e iniciado.

O código acima se parece com um código HTML normal e aqui a tag <title> mostra o mesmo título no navegador que o título do aplicativo. Mas o código

<body> é diferente do código HTML normal. Aqui, você vê a tag "<app-root>", fornecida pela CLI. Podemos dizer que, sempre que criamos um projeto a partir da CLI, por padrão, um componente é criado, ou seja, "componente do aplicativo".

Agora, consulte o arquivo "app.component.ts". É um arquivo TypeScript. Aqui, você vê a propriedade "selector".

Você pode ver que a propriedade seletor contém a sequência como arquivo index.html. Essas informações são necessárias para que o Angular coloque essa parte em um arquivo index.html com o modelo do componente.

O modelo do componente é "./app.component.html", portanto, o Angular inclui essa parte no corpo do arquivo index.html.

Agora, você vê como uma "raiz do aplicativo" está incluída no arquivo index.html.

Beleza, vamos ver como o Angular é acionado?

Sempre que o ng-serve cria o aplicativo, ele cria "pacotes" e os adiciona automaticamente ao arquivo index.html em tempo de execução. Portanto, a partir desses pacotes configuráveis, o primeiro código deve ser executado a partir do arquivo "main.ts", ou seja, o arquivo "main.ts" é o arquivo principal de onde a execução de um aplicativo Angular é iniciada.

Aqui, o método de autoinicialização inicia o aplicativo Angular. Refere-se ao AppModule, que analisa as pastas do aplicativo. Você pode ver no arquivo "app.module" que uma matriz de autoinicialização que é basicamente uma lista de todos os componentes analisa o arquivo index.html.

Agora, você pode ver que o aplicativo Angular é carregado como:

main.ts >> app.Module.ts >> app.component.ts >> index.html >> app.component.html
