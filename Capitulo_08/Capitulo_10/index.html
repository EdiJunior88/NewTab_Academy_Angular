Componentes

A maior parte do desenvolvimento com o Angular é feita nos componentes. Agora de forma mais direta, componentes são basicamente classes que interagem com o arquivo .html do componente, que é exibido no navegador. Vimos a estrutura do arquivo em um de nossos conteúdos anteriores.

Então, a estrutura do arquivo tem o componente do aplicativo e consiste nos seguintes arquivos:

    app.component.css
    app.component.html
    app.component.spec.ts
    app.component.ts
    app.module.ts

E se você selecionou o roteamento angular durante a configuração do projeto, os arquivos relacionados ao roteamento também serão adicionados e os arquivos serão:

    app-routing.module.ts

Os arquivos acima são criados por padrão quando criamos um novo projeto usando o comando angular-cli.

O angular-cli possui um comando para criar seu próprio componente. No entanto, o componente do aplicativo criado por padrão sempre permanecerá o pai e os próximos componentes criados formarão os componentes filhos.

Vamos agora executar o comando para criar o componente com a linha de código abaixo:

    ng g component new-cmp
